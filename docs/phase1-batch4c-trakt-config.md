# Batch 4c (TraktConfig plumbing + Trakt repos + CollectionEditorRepository)

Unblocks the Trakt repository cluster that 4b deferred, using the same generated-config plumbing
pattern Batch 1 used for the Supabase config.

## The unlock: TraktConfig into :shared

`TraktConfig` (`object TraktConfig { CLIENT_ID, CLIENT_SECRET, REDIRECT_URI }`) was generated by
composeApp's runtime-config task into composeApp's build dir only — so any `:shared` file
referencing it failed to resolve (the trap that surfaced in 4b, same as Batch 3's
`ImdbEpisodeRatingsConfig`).

Fix (mirrors Batch 1's Supabase/version config move):
- **shared/build.gradle.kts**: added 3 inputs (`traktClientId/Secret/RedirectUri`) +
  a `com/nuvio/app/features/trakt/TraktConfig.kt` generation block to
  `GenerateSharedRuntimeConfigsTask`; wired from `sharedRuntimeConfigValue("TRAKT_CLIENT_ID"/
  "TRAKT_CLIENT_SECRET"/"TRAKT_REDIRECT_URI")` (redirect defaults to `nuvio://auth/trakt`).
- **composeApp/build.gradle.kts**: removed the trakt generation block; replaced with a
  `.delete()` of any stale composeApp-generated `TraktConfig.kt` (avoids duplicate FQN — both
  modules generating the same `com.nuvio.app.features.trakt.TraktConfig` would clash). composeApp
  gets it transitively via `implementation(projects.shared)`.

## Moved to :shared (6 files, all 4b-reverts)

- **trakt:** TraktAuthRepository (TraktConfig), TraktCommentsRepository (→TraktAuthRepository),
  TraktRelatedRepository (→TraktAuthRepository), TraktAuthBridge (→TraktAuthRepository),
  TraktPublicListSourceResolver (TraktConfig + CollectionModels' TraktListSort/SortHow, moved 4b).
- **collection:** CollectionEditorRepository (→TraktPublicListSourceResolver + CollectionModels +
  TmdbCollectionSourceResolver, all moved). Its i18n was already decoupled in 4b (resourceString).

All 6 verified: no external blockers, no same-package deferred refs, no other generated configs,
no i18n leftover, no internal symbols consumed by staying composeApp (nothing to widen), no
duplicate FQN, no reverse-leak imports.

into `:shared`. Possible composeApp smart-casts on CollectionSource/Collection re-accesses in
staying screens.
